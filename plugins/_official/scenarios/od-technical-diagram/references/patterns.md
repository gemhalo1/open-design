# Diagram Patterns

- Architecture: client, edge/API, services, workers, stores, observability.
- Workflow: lanes, triggers, states, decision branches, exceptions, outputs.
- RAG/agent: ingest, embed, vector store, retriever, model, tools, memory, eval loop.
- UML sequence: lifelines, activations, request/response, retry/error path.
- Data flow: sources, transforms, warehouse, semantic layer, dashboards, governance.
- Comparison: current vs proposed, before/after, decision matrix, tradeoff quadrant.
- Infographic: title, key stat, supporting evidence, legend, takeaways.

Use the simplest grammar that explains the relationship. Avoid diagram soup.
